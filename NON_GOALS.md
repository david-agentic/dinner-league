# NON_GOALS.md — what Dinner League refuses to be

The most important document, because almost nobody writes it.
A philosophy with no "no" is an accelerator with cut brake lines.

## We refuse to become
- Accounting software / ERP / corporate dashboard
- An expense tracker (Splitwise, Excel, QuickBooks)
- A social network or messenger (WhatsApp already exists — we FEED it, never replace it)
- A marketplace, a payments processor, or anything crypto

## We will NOT
- Require more than a handful of taps to log dinner
- Add a feature just because a competitor has it
- Hide balances behind animations, or make money hard to read
- Sacrifice speed for visual effects
- Force daily engagement with guilt or manipulation (dark patterns)
- Build gambling mechanics that move real balances
- Integrate real payment rails (JazzCash/Easypaisa settle-now) — regulatory + fraud swamp for a solo founder. Manual "mark settled" only.
- Reset anyone's real money when a season resets. Glory resets; money never does.

## The 10-Second Rule is non-negotiable
Any feature that slows logging dinner is wrong by default. The core loop is sacred.
Everything else orbits it.

## KILL LIST (living section — review monthly)
Great software is built by adding, testing, learning, and *removing*.
Each month, ask not "what should we improve?" but "what should we DELETE?"

Candidates get listed here with a date and a reason. If a feature has been live a
month and the group doesn't touch it, it goes on this list, then it goes.
